======================================

This file documents all the major user prompts and development phases used to create the RouteIQ application.
The app is a route optimization and risk analysis system with modern UI and Geotab fleet telematics integration.

TECHNICAL ARCHITECTURE
======================

Backend: FastAPI + Python
- OSRM API: Multiple route alternatives (fastest + safe)
- Geotab Integration: Fleet telematics with JSON-RPC API
- Incident Detection: Multi-criteria system
  * Aggressive Speeding: >20% over speed limit
  * Harsh Braking: ≥15 km/h deceleration
  * Rapid Acceleration: ≥15 km/h acceleration
  * Sharp Turns: >60 degree bearing change
  * Slow Driving: <50% of speed limit
- Threading: Prevents frontend freeze during incident data fetching
- Timeout: 15-second limit on incident collection with intelligent backoff

Frontend: React/Next.js + Modern Design
- UI Design: Purple-pink gradient (#667eea → #764ba2)
- Glassomorphism effects with backdrop blur
- Navigation bar with active page detection
- Multi-page structure: Home, Risk Analysis, Heatmap, Fleet
- Responsive design with Tailwind CSS
- Leaflet-based interactive maps

Key Features:
- Risk scoring based on incident frequency and type
- Day-based time filtering with timezone awareness
- GPS heatmap visualization with gradient concentration
- Multi-vehicle fleet analysis and tracking
- Bearing calculation for dangerous turn detection
- Incident clustering (100-meter grid accuracy)
- Top 50 hotspots identification and ranking

INCIDENT DETECTION CRITERIA
===========================
- Aggressive Speeding: Vehicle travels >20% over the legal speed limit
- Excessive Slow Driving: Vehicle travels <50% of the speed limit on highways (zones >60 km/h)
- Harsh Braking: Speed reduction of ≥15 km/h in a short time period
- Rapid Acceleration: Speed increase of ≥15 km/h in a short time period
- Sharp Turns: Bearing angle change >60 degrees while traveling at speeds >30 km/h

RISK SCORING METHODOLOGY
=======================
- Incident-based ranking: Risk scores calculated from actual incident frequency at each location
- Spatial clustering: All incidents grouped into 100-meter grid cells for hotspot identification
- Ranked by severity: Locations sorted by total incident count detected in the timeframe
- Visual concentration: Gradient colors indicate incident density (green=low, yellow=moderate, red=high)
- Top hotspots: Up to 50 highest-risk locations displayed on the heatmap
